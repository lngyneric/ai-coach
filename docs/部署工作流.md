# 部署工作流

## 原则

1. 所有代码修改先在 dev 环境验证
2. 生产环境只能通过 Git 标签 + 镜像构建部署
3. 禁止直接 `docker cp` 到生产容器

---

## 分支策略

```
main          ← 日常开发，可随意修改
release/*     ← 发布候选，需 Code Review
production    ← 生产分支，仅从 release 合并
```

**流程：**
```
dev 验证通过
   ↓ 提交 main
   ↓ 创建 PR → release/v2.x.x
   ↓ Code Review → 合并
   ↓ 打 tag v2.x.x
   ↓ 构建镜像 → docker compose up -d
```

---

## 环境配置

| 文件 | 用途 |
|------|------|
| `docker/.env` | 开发环境配置 |
| `docker/.env.prod` | 生产环境配置（独立密钥、独立 DB） |

**生产环境配置关键差异：**
```diff
- AAD_BYPASS=1
+ AAD_BYPASS=          # 关闭 bypass，强制 AAD 认证

- SECRET_KEY=开发密钥
+ SECRET_KEY=<安全随机密钥>  # python3 -c "import secrets; print(secrets.token_urlsafe(32))"

- LOG_LEVEL=info
+ LOG_LEVEL=warning          # 减少日志量
```

## 部署步骤

### 1. 代码修改 → Dev 验证
```bash
# 在 dev 环境修改代码
cd /home/sysmex/ai-shifu

# 前端修改：dev 容器自动热重载
# 后端修改：docker cp + gunicorn SIGHUP
./deploy.sh
```

### 2. 验证通过 → 提交代码
```bash
git add .
git commit -m "feat: xxx"
git push origin main
```

### 3. 创建 Release
```bash
# 创建 release 分支
git checkout -b release/v2.0.2 main
git push origin release/v2.0.2

# Code Review... 合并到 production
git checkout production
git merge release/v2.0.2
git tag v2.0.2
git push origin production --tags
```

### 4. 构建并部署生产镜像
```bash
# 构建 API 镜像
cd /home/sysmex/ai-shifu/src/api
docker build -t aishifu/ai-shifu-api:v2.0.2 .
docker tag aishifu/ai-shifu-api:v2.0.2 aishifu/ai-shifu-api:latest

# 构建前端镜像
cd /home/sysmex/ai-shifu/src/cook-web
docker build -t aishifu/ai-shifu-cook-web:v2.0.2 .
docker tag aishifu/ai-shifu-cook-web:v2.0.2 aishifu/ai-shifu-cook-web:latest

# 更新 docker-compose.yml 版本号
cd /home/sysmex/ai-shifu/docker
sed -i "s|image: aishifu/ai-shifu-api:.*|image: aishifu/ai-shifu-api:v2.0.2|" docker-compose.yml
sed -i "s|image: aishifu/ai-shifu-cook-web:.*|image: aishifu/ai-shifu-cook-web:v2.0.2|" docker-compose.yml

# 部署
docker compose up -d
```

### 5. 验证生产
```bash
curl -I https://eu.sysmex.com.cn/
curl http://10.32.16.47:8080/api/runtime-config
```

## 禁止操作

| ❌ 禁止 | ✅ 改为 |
|---------|---------|
| `docker cp` 到生产容器 | 构建新镜像 → `docker compose up -d` |
| 直接修改生产 `route.py` | 改源码 → 提交 → 构建镜像 |
| 修改 `.env` 后不提交 | `.env.prod` 修改需提交 git（不含密钥） |
| 在生产容器内执行 `npm install` | 重建前端镜像 |
