# 课程结构说明

## 1. 基本概念

```
课程（Shifu）              ← 一个完整的培训课程
 ├── 章节（Chapter）        ← type=1 / level 0，课程大纲分组
 │    ├── 课时（Lesson）    ← type=2 / level 1，AI 互动课
 │    └── 课时（Lesson）    ← type=2，含 {{video}} 标记的视频课
 └── 章节
      └── ...
```

## 2. 数据库表结构

### 2.1 课程主表 `shifu_published_shifus`

| 字段 | 说明 | 示例 |
|------|------|------|
| `shifu_bid` | 课程唯一 ID | `2136209a044041e99d78973d3c75fa86` |
| `title` | 课程名称 | AI 新经理操作指南 |
| `description` | 课程简介 | |
| `keywords` | 逗号分隔的关键词 | `领导力,视频` |
| `cover_res_bid` | 封面图资源ID | |
| `llm` | 使用的LLM模型 | `deepseek-v4-flash` |
| `state` | 发布状态 | 1101=草稿, 1102=已发布 |

### 2.2 课时内容表 `shifu_published_outline_items`

| 字段 | 说明 | 常见值 |
|------|------|--------|
| `outline_item_bid` | 课时 ID | |
| `shifu_bid` | 所属课程 | |
| `title` | 课时标题 | |
| `type` | 类型 | `1`=章节, `2`=课时 |
| `parent_bid` | 父章节 ID | 属于哪个章节 |
| `position` | 排序号 | `0101` |
| `content` | mdflow 内容 | 见下方格式说明 |

| `ask_enabled_status` | 追问开关 | `5101`=关闭, `5102`=开启 |

## 3. mdflow 内容格式

### 3.1 普通 AI 互动课时

标准 Markdown 格式，系统会自动渲染并启动 AI 对话：

```markdown
# 课时标题

这里是课程正文内容。支持 Markdown 语法：

- 列表
- **加粗**
- 引用

---

**互动环节：**
1. 问题一
2. 问题二
```

系统处理流程：
1. 加载 mdflow 内容 → 渲染为课件页面
2. 自动调用 AI 对话（使用配置的 LLM 模型）
3. 学习者可提问、追问
4. 生成学习记录

### 3.2 视频课时格式

第一行必须是 `{{video}}`，第二行是视频 URL（HLS `.m3u8` 或 MP4）：

```markdown
{{video}}
/vod/304e2221703571f182df4531959d0102/9cd7112d30a54824bf074039d24cd1a8-325ad47ab11af91da77d997fdde6786f-sd.m3u8
```

系统处理流程：
1. 检测到 `{{video}}` → 跳过 AI 渲染
2. 跳转到独立视频播放页面 `/video-player.html?bid={shifu_bid}`
3. 使用 hls.js 播放 HLS 流（支持 `https://video.sysmex.com.cn/...` 或 `/vod/...` 代理路径）
4. 无 AI 交互，仅播放视频
5. 播放进度记录到 `course_enrollments.progress_pct`

## 4. 课程分类体系

### 4.1 分类方式

通过 `shifu_published_shifus.keywords` 字段管理，用逗号分隔：

| 关键词 | 含义 | 示例 |
|--------|------|------|
| `领导力` | 领导力课程 | `领导力,视频,必修` |
| `学术` | 学术/专业课程 | `学术.检验,视频` |
| `产品` | 产品培训 | `产品,视频` |
| `视频` | 视频课程（触发视频播放器） | `领导力,视频` |

### 4.2 培训模块映射（training-portal）

| 模块标识 | 显示名称 | 默认按钮 |
|----------|----------|----------|
| `onboarding` | 🎓 新人培训 | ✅ |
| `mentorship` | 📚 学分制带教 | ✅ |
| `intensive` | 🔥 小灶培训 | ✅ |
| `leadership` | 👑 领导力课程 | ✅ |

映射关系由 `course_enrollments.module` 字段决定，管理员分配时选择。

### 4.3 API 端点

| 端点 | 用途 |
|------|------|
| `GET /api/shifu/shifus` | 课程列表（支持 keywords 过滤） |
| `GET /api/shifu/enrollments` | 当前用户已分配课程（含进度） |
| `GET /api/shifu/dashboard` | 学习统计（总数/完成/进度） |
| `POST /api/shifu/admin/enroll` | 管理员分配课程 |
| `GET /api/shifu/admin/enrollments` | 管理员查看分配清单 |
| `GET /api/shifu/admin/operations/users` | 管理员查看用户列表 |

## 5. 课程创建流程

```
管理员
  ├── 创建课程（填写标题、描述、关键词）
  │    ├── 添加章节（type=1）
  │    │    └── 添加课时（type=2）
  │    │         ├── 普通课时 → 编写 mdflow → AI 渲染
  │    │         └── 视频课时 → 写入 {{video}}\nURL → 视频播放器
  │    └── 发布课程
  │
  └── 分配课程（training-portal 管理界面）
       ├── 选择学员
       ├── 选择课程
       └── 选择模块（新人培训/学分制带教/小灶培训/领导力）
```
