## 一、平台入口

### 1.1 访问地址

| 环境 | 地址 |
|------|------|
| 外网（公网） | `https://eu.sysmex.com.cn/courses/` |
| 内网 | `https://10.32.16.47/courses/` |
| 课程播放器 | `https://eu.sysmex.com.cn/c/{课程bid}` |
| 课程编辑 | `https://eu.sysmex.com.cn/shifu/{课程bid}` |

### 1.2 登录方式

首次访问 `/courses/` 会自动跳转到登录页：

- **工号**: 如 `sch0000` 格式
- **密码**: 首次登录请联系 HR 获取初始密码
- **登录后可保持登录状态 24 小时**

### 1.3 登出

点击页面右上角的 **"退出"** 按钮，系统会：

1. 清除登录凭证（Cookie + localStorage）
2. 自动跳转到登出完成页 `/logout.html`
3. 可选择「重新登录」或「返回课程首页」

## 二、课程首页（学员视图）

### 2.1 页面布局

访问 `/courses/` 后，页面从上到下分为：

```
导航栏（Logo + 用户名 + 退出）
欢迎区（时段问候 + 导师工作台入口*）
统计卡片（在学课程 / 已完成 / 学习时长 / 积分）
学员看板（进度概览 + 待办事项 | 成长曲线 + AI建议）
四大分类卡片（新人入学 / 学分制带教 / 小灶教学 / 领导力课程）
分类课程列表
页脚
```

> *如果你是导师（名下有待教学员），欢迎区会显示「🧑‍🏫 导师工作台」入口

### 2.2 四大课程分类

| 分类 | 适用人群 | 特点 |
|------|----------|------|
| 🎓 **新人入学** | 新入职普通员工 | 入职引导、组织架构、基础业务流程 |
| 📚 **学分制带教** | 一线营业/初级员工 | 系统化导师带教、分阶段验收、学分制管理 |
| 🔥 **小灶教学** | 一线营业/二线支援 | 精品小班、案例分析、模拟演练 |
| ⭐ **领导力课程** | 管理层 | 团队管理、战略思维、决策能力 |

**操作**: 点击分类卡片可筛选只看该分类的课程，再次点击取消筛选。

### 2.3 课程卡片

```
┌──────────────────────┐
│ 🔵 课程名称          │
│ 课程简介文字（最多   │
│ 显示两行）           │
│ 📖 阅读 / 🎧 语音    │
└──────────────────────┘
```

- **点击卡片** → 进入课程播放器 `/c/{bid}`
- 🎧 **语音** = 该课程支持 TTS 语音朗读
- 📖 **阅读** = 纯文字阅读

展示近 6 个月的课程完成趋势柱状图。

基于学习记录和岗位信息，动态推荐下一步学习内容。

### 3.6 通知

右上角可查看未读通知。系统会自动推送：
- 新任务分配通知
- 阶段开启/截止提醒
- 评分完成通知
- 转正提醒

## 四、学习流程

### 4.1 学员标准路径

```
1. 登录 /courses/ 
2. 选择分类或浏览全部课程
3. 点击课程卡片 → 进入课程播放器
4. 按章节学习（阅读/语音）
5. 完成课后测验/交互
6. 在学员看板查看进度更新
```

### 4.2 学分制带教路径（新员工完整流程）

```
入职 → 版块一：入职培训（3天内）
         ├─ HR：入职手续/合同/组织架构
         ├─ 上司：责任田/政策/MBO/带教计划
         └─ 助理：CRM/报销/会议/战略报告
         ↓ 互评报告
    → 版块二：小灶培训（1月内，可选）
         ↓ 互评报告
    → 版块三：集中培训（3月内）
         ├─ E-learning 扫盲自学
         ├─ 上海 3 天培训营
         └─ 结业考试
         ↓ 扫盲考 + 结业考通过
    → 学分制带教第一阶段（1-2月）
         ├─ 血球/尿液/凝血产品培训
         ├─ 市场政策学习
         ├─ 任务数字确认
         └─ 代理商/客户拜访带教
         ↓ 全部验收通过
    → 学分制带教第二阶段（1-2月）
         ├─ 免疫/生化/糖化/Alifax
         ├─ 提案培训
         ├─ 渠道谈判带教
         └─ KOL 拜访带教
         ↓ 全部验收通过
    → 学分制带教第三阶段（1-2月）
         ├─ 生命科学产品培训
         ├─ 招标培训
         ├─ 压货谈判带教
         └─ 现场招标带教
         ↓ 全部验收通过
    → 学分汇总 → 转正判定
```

### 4.3 转正标准

| 总学分 | 结果 |
|--------|------|
| ≥ 85 分 | ✅ 正常转正 |
| 70 ~ 84 分 | ⏳ 延长考核一个月 |
| < 70 分 | ⚠️ 延长两个月 / 劝退 |

### 4.4 评分标准

| 维度 | 权重 | 评分人 |
|------|------|--------|
| 理论考试 | 40% | HR |
| 现场实操 | 30% | 带教经理/导师 |
| 互评报告 | 20% | 自评20%+带教人50%+总监30% |
| 带教经理评价 | 10% | 带教经理 |

### 5.3 导师核心操作

| 操作 | 方式 | 说明 |
|------|------|------|
| 查看学员进度 | 导师工作台直接查看 | 每位学员显示当前阶段状态 |
| 待评分提醒 | 红色角标数字 | 需要导师评分的验收项数量 |
| 提交评分 | 通过 API（前端开发中） | `POST /api/portal/mentorship/items/{bid}/score` |
| 创建任务 | 通过 API（前端开发中） | `POST /api/portal/tasks` |

### 5.4 导师职责

1. **为新员工制定带教计划**：入职 3 天内完成版块一培训安排
2. **跟进学习进度**：定期检查学员看板，关注阶段截止提醒
3. **评分与反馈**：学员提交验收项后 **48 小时内** 完成评分
4. **定期沟通**：每周至少 1 次 1v1 沟通

## 六、管理员操作指南

### 6.1 学员管理

**新增学员档案**（通过 API）：

```bash
curl -X POST /api/portal/admin/learners \
  -H "Content-Type: application/json" \
  -d '{"user_bid":"xxx", "employee_no":"sch0001", "department":"销售部", "position_name":"区域销售经理"}'
```

**编辑学员**：

```bash
curl -X PUT /api/portal/admin/learners/{learner_bid} \
  -d '{"mentor_bid":"xxx", "level":"管理层"}'
```

**查看列表**：

```bash
GET /api/portal/admin/learners?page=1&size=20
```

### 6.2 带教阶段配置

**查看阶段配置**：

```bash
GET /api/portal/admin/phases
```

**修改阶段参数**（时长/权重/及格线）：

```bash
curl -X PUT /api/portal/admin/phases/{phase_bid} \
  -d '{"duration_days":45, "passing_score":70}'
```

### 6.3 验收项管理

**查看阶段验收项**：

```bash
GET /api/portal/admin/checklist/{phase_bid}
```

**新增验收项**：

```bash
curl -X POST /api/portal/admin/checklist \
  -d '{"phase_bid":"xxx", "name":"血球产品三个明白考试", "category":"exam", "max_score":5}'
```

category 可选值：exam / practice / review / mentor，对应评分四维度。

### 6.4 课程分类管理

通过 `shifu_category_map` 表将课程归类：

```sql
INSERT INTO shifu_category_map (shifu_bid, category_bid)
VALUES ('{课程bid}', 'cat-006');
```

| slug | 名称 | 说明 |
|------|------|------|
| onboarding | 新人入学 | 新员工入职 |
| mentorship | 学分制带教 | 带教类课程 |
| intensive | 小灶教学 | 实战精品 |
| leadership | 领导力课程 | 管理层发展 |

### 6.5 开启带教阶段

为学员开启一个带教阶段：

```bash
curl -X POST /api/portal/mentorship/start \
  -d '{"learner_bid":"xxx", "phase_bid":"xxx"}'
```

系统会自动创建 `learner_mentorship` 记录并推送通知给学员。

### 6.6 全局统计

```bash
GET /api/portal/admin/stats
# 返回: total_learners, active_learners, in_progress_mentorships, passed_mentorships
```

## 八、常见问题

### Q1: 登录后看不到课程？

A: 请确认账号已被分配课程。管理员可检查 `shifu_category_map` 和 `ai_course_auth` 表。

### Q2: 课程分类不对？

A: 当前基于课程名称和描述的关键字匹配。管理员可在 `shifu_category_map` 表绑定课程到具体分类。

### Q3: 带教阶段状态不对？

A: 联系管理员检查 `learner_mentorship` 表中的状态字段。

### Q4: 没有看到"聆听"模式？

A: 课程详情页右上角有「阅读/聆听」切换按钮，需要该课程启用了 TTS（课程卡片显示 🎧 语音标签）。

### Q5: 忘记密码？

A: 联系 HR 重置密码。

### Q6: 如何查看自己的学习记录？

A: 登录 `/courses/` 在学员看板查看，包含进度、待办、成长曲线等。

### Q7: 收到评分通知但无法评分？

A: 确认你是该学员的 `mentor_bid`。只有被分配的导师才能对验收项评分。

### Q8: 导师工作台没有显示？

A: 只有当你的账号被设置为至少一个学员的 `mentor_bid` 时，导师工作台入口才会出现。联系 HR 检查 `learner_profiles` 表的配置。
