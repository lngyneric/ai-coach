# 系统交互按钮例外处理

## 适用场景

- 聊天区或课件区的 `interaction` 内容里包含系统按钮值，例如 `_sys_pay`、`_sys_login`、`_sys_next_chapter`。
- 第三方渲染层会根据 `user_input`、`defaultButtonText` 或 `readonly` 把系统按钮渲染成已提交态，导致按钮变蓝、禁用或无法再次点击。

## 处理原则

- 不修改 `markdown-flow-ui` 源码时，优先在 `ai-shifu` 的包装层识别系统按钮内容，再覆写传给渲染器的 `readonly` 和默认值。
- 对需要重复触发的系统按钮，禁止把历史 `user_input` 回灌成默认选中态，避免按钮渲染成蓝色已选中状态。
- 若第三方组件内部仍可能命中默认 `.select` 或 `:disabled` 样式，补一层项目侧 CSS 覆写，确保按钮颜色和可点击状态符合业务预期。
- 例外逻辑只作用于目标系统按钮，不要影响普通问答交互、反馈交互或多选交互。

## 验证要点

- 首次点击系统按钮后，按钮视觉保持白色，不进入蓝色选中态。
- 关闭相关弹窗或中断流程后，按钮仍可再次点击。
- 普通 interaction 的默认回填、只读态和提交态样式保持不变。
