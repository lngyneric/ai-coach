# 实时洞察：把"事后烟"变成"天气预报"

用一个真实的决策困境开场——某部门提出 3 名员工转正申请，HR 手头只有考勤记录和一份主观评价表。没有横向对比、没有趋势数据、没有风险预警。等转正后发现其中一名员工能力不匹配，已经过了试用期，企业只能接受低效或走裁员流程。

Ask the learner to calibrate: 你们公司最近一次转正评估，决策依据是什么？请选最接近的一项。

?[%{{decision_basis}} 主管一句话 | 考勤 + 主观评价 | 有简单的考核表 | 我们有完整的评估体系]

基于 {{decision_basis}} 回复。如果选择"主管一句话"，不用评判，而是指出现实——大多数中小企业都是这样，这恰恰是 AI-Coach 要改变的地方。如果有考核表，指出单次评估的盲区——没有历史数据（趋势）、没有横向数据（对比）、没有预测数据（风险）。

向学习者介绍 AI-Coach 的分析报告核心模块：

**阶段成绩趋势图（echarts 可视化）**
- 多阶段总分趋势线 + 及格线 markLine（虚线标注）
- 一眼看到学员是"稳定上升"、"波动大"还是"持续不达标"
- 数据自动更新，不需要 HR 手动录入

**能力维度构成（分组柱状图）**
- 理论 / 实操 / 互评 / 导师评价四维并排对比
- 发现"偏科"——理论满分但实操挂零的学员
- 支持多学员横向对比（团队效能分析）

**面谈时间线**
- 每次面谈的日期、类型、主题、笔记摘要、行动项
- AI 自动汇总面谈趋势（比如"最近 3 次面谈都聚焦同一问题 → 需要额外辅导"）

Explain the boundary: 当前版本的分析报告基于学员已有的验收数据和面谈记录。AI 的"预测"本质上是趋势推断——如果前两个阶段持续未达标，第三个阶段大概率也如此。这不是占卜，而是统计学。

Ask the learner: 如果明天就能看到一份完整的员工成长报告，你最想先看到什么？

?[%{{report_priority}} 该员工是否适合转正 | 整个团队的培训效能排名 | 哪些人在离职风险区 | 培训投入产出比]

基于 {{report_priority}}，用具体场景展开 AI-Coach 如何满足这个需求。"转正评估"→ 多维度对比 + 趋势判断；"团队排名"→ 横向比较 + 亮点/短板标注；"离职风险"→ 未达标预警 + 建议辅导计划。

总结：实时洞察 = 历史数据 + 横向对比 + 趋势预测。它不替代管理层的决策，但让决策不再"拍脑袋"。行动建议：先在现有转正流程中增加一步——要求主管提交一份 AI-Coach 生成的学员成长报告，作为转正评估的参考附件。
